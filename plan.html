Phase 2 — Prisma Migration

Migration Strategy

Multi-step migration to rename/replace enums safely:

  1. Create new ProjectStage enum with 10 values
  2. Add new columns (stage, priority, region, notification, etc.)

  7. Set NOT NULL constraints on required fields

Phase 6 — Excel Import Feature Adjustments

The current import service expects a custom CSV/Excel template with English column names. It needs to be rewritten to consume the PLN "ALL" sheet format directly.

6.1 Backend: import.service.ts — Sheet & Header Handling

Current behavior
  • Reads the first sheet of the uploaded file
  • Assumes Row 1 = header with English column names (name, type, status, etc.)
  • Maps 3 statuses via STATUS_MAP, 3 types via TYPE_MAP
New behavior
  • Read the "ALL" sheet by name (fallback to first sheet if not found)
  • Row 1 (index 0) is a count/summary row — SKIP
  • Row 2 (index 1) is the actual header with Indonesian column names
  • Data starts from Row 3 (index 2)
  • Only read columns 0–24 — columns 25–43 are helper/internal columns → IGNORE

6.5 Backend: import.service.ts — parseBuffer() Update

Current
const wb = XLSX.read(buffer, { type: 'buffer' });
const ws = wb.Sheets[wb.SheetNames[0]];  // always first sheet
return XLSX.utils.sheet_to_json(ws, { defval: '' });
New
const wb = XLSX.read(buffer, { type: 'buffer' });
// Prefer "ALL" sheet, fallback to first
const ws = wb.Sheets['ALL'] ?? wb.Sheets[wb.SheetNames[0]];
// Row 1 = summary counts, Row 2 = headers → use range starting from row 2
const range = XLSX.utils.decode_range(ws['!ref']);
range.s.r = 1;  // skip row 0 (summary), use row 1 as header
ws['!ref'] = XLSX.utils.encode_range(range);
return XLSX.utils.sheet_to_json(ws, { defval: '' });
